Category: Algorithms & Data Structures Course: CS-499

Brief Description

Enhancement 2 focused on implementing a fucntional slideshow similar to the original project but implemented a sorting method to display destinations by their overall rank. It utilizes a simple scrolling method to allow a user to navigate through the page without any refreshing required. This enhancement also implemented a hashmap to allow fast lookups of data by their name.

Enhancement Narrative

Briefly describe the artifact. What is it? When was it created?

The artifact I chose for this enhancement was a slideshow originally created in COS 250. It is a slideshow that uses a simple Java program to display a top 5 destination slideshow. It uses JFrame to accomplish this which is considered legacy within Java itself. It has a lot of limitations to it being simple because everything is hard coded into the program, this was one of my earliest programming assignments and it does not include any algorithms or data structures within the program. The enhanced version implements both of these things within a Node.js and Express web application.

Justify the inclusion of the artifact in your ePortfolio. Why did you select this item? What specific components of the artifact showcase your skills and abilities in software development? How was the artifact improved?

This artifact was selected because it has a lot of potential for growth and development. Its simplicity made it an ideal foundation for expanding upon what I’ve learned. The original Java application has none of the requirements needed for enhancement 2, it does not utilize data structures or any algorithms and it simply cycles through a hard coded list of images. By converting it to a web application and adding a computational layer I was able to expand the complexity and modularity of the project.

This enhancement effectively showcases my applied skills in algorithms and data structures. First I implemented a dedicated algorithm that provides a way to dynamically filter and sort the destination data. The sorting logic I selected to accomplish this leverages JavaScript’s Timsort based .sort(), this at a worst case scenario is o(n log n) for a time complexity and when filtering only uses a time complexity of o(n) in a linear scan to find partial location matching or the rating. On top of this method to sort the data I added a map based lookup that allows a user to look up data with O(1) complexity by ID, this bypasses the O(n) scanning approach when applicable. These enhancements demonstrate an understanding of performance tradeoff and an appropriate structure selection.

Reflect on the process of enhancing and modifying the artifact. What did you learn as you were creating it and improving it? What challenges did you face?

This enhancement was more about critical thinking, at first I was not sure what algorithm I wanted to use for the overall data manipulation. I originally was going to use a quick sort to accomplish the sorting of the data but I learned that JavaScript’s .sort() uses a Timsort-based approach. The algorithm I wanted to use in the best case scenario was a time complexity of O(n log n) which is acceptable but Timsort has a worst case scenario of the same time complexity. The sorting was performed using JavaScript’s Timsort-based .sort() implementation, because in a professional scenario an employer would want you to use the more optimized sort method which reduces the overall amount of resources needed to compute the data. Although this would not be noticed much with small datasets it becomes a problem as the datasets get larger. Another thing I ran into in this enhancement was when creating the data structures I wanted to make it so I could easily integrate MongoDB in the future, this made it feel more like a team environment because I was working with future me to create the scaffolding of enhancement three so the 2 can be put together like a puzzle. This made me think more of the long term in terms of development. My choices were intentional and focused on supporting future scalability. This was also one of the challenges I faced with these enhancements, I was thinking to myself what is the best way to handle this without changing a lot of the logic from enhancement 2. Overall I decided to have a controller that just switches to the use of MongoDB and as a failsafe uses the JSON as a backup.
